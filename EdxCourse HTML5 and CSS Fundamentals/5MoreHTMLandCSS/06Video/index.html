Poster: Specifies the URL of the frame you want to display as the video cover until the user starts or seeks the video. By default, the first frame is considered the poster frame. The poster can also be an arbitrary image, not necessarily in any frame of the video. Preload: Allows author to communicate to the browser which settings will work best - video should not be preloaded (none), only video metadata is fetched (metadata), video file can be downloaded (auto) values: none, metadata, auto

Just like audio file formats, not all video file formats are supported by all browsers. You should use common video file formats for browser compatibility ensuring the highest probability that your video file will play. The most common ones are MP4, WebM and Ogg. type specifies the internet media type

Source element is also used to specify multiple source files for the video element.
