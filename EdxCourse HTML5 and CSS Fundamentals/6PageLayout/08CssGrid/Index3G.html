Quick Tip: Name Your CSS Grid Lines, Just in Case
Grid module allows us to explicitly name our lines when we declare our grid columns and rows.
we can wrap our column and row values with line names (whatever the heck we want), using square brackets: grid-template-columns: [start] 100px [centre-start] auto [last-column-start] 100px [finish];
1

Some Pointers

  1. These names can be tailored to your own descriptive needs, so think logically about what will help you recognize and remember areas of the grid.
  2. The original line numbers remain in operation, so you can still use them.
  3. You can declare multiple names for one line, for example: [main-end footer-start row-5] etc.
  4. Even if you name your grid lines, you’re not obliged to use them, so it’s a good “just in case” habit to get into.
There are also circumstances where lines are given names implicitly.
Naming a grid area header automatically assigns names to its four boundary lines too. The row lines around it become header-start and header-end, and likewise the two column lines also become header-start and header-end. The same applies to the other areas
Note: Looking at things in reverse, explicitly adding named lines in the same format (header-start and header-end) will create a named grid area of header.
We can use these line names just like we have previously, to position items. And again, they exist in addition to any names you define yourself, and the original line index numbers.
